# 项目执行规范

## 重构后的工作提示词

> 以“完成本组《发电厂电气部分课程设计》任务书并形成可提交、可复核、可持续维护的工程成果”为总目标。先完整盘点项目文件、任务书、模板、基础数据与既有图纸；建立公开 GitHub 工程仓库，但在公开前排除个人隐私、账号信息、锁文件、缓存、大型无关附件及不宜公开的材料。随后输出依赖有序的总体计划，覆盖原始资料整理、设计计算、设备选择与校验、说明书编制、电气主接线简图、配电装置断面图、配电装置平面布置图、CAD 出图规范、质量审查、最终交付和版本同步。所有关键结论必须能追溯到任务书、计算过程、标准规范或图纸；所有后续修改均记录变更、验证后提交并同步 GitHub。遇到组别、原始参数、学校模板、CAD 版本或公开边界不明确时，先从本地材料查证，确实无法判断再向用户提问。

## 成功标准

项目只有在以下条件同时满足时才算完成：

1. 任务书中的每项内容都有明确产物，并能在需求追踪矩阵中定位。
2. 说明书、计算书、设备表和三张图使用同一组参数，不互相矛盾。
3. 每个设备选择均有额定电压、持续电流、开断、动稳定和热稳定依据。
4. CAD 图纸可编辑、可打印、字体不缺失，并导出兼容版 DWG 与 PDF。
5. 原始资料和公开成果严格分离，Git 历史中不出现隐私或第三方受限文件。
6. 每个有效修改批次完成检查、日志、提交、推送和远端验证。

## 工作边界

- 不把参考设计书直接改名为本项目成果。
- 不在缺少设备尺寸和安全净距时提前定稿断面图或平面图。
- 不把未经确认的设备型号、标准条文或计算系数写成既定事实。
- 不把 CAD/Office 自动保存文件、提交压缩包或本地环境写入仓库。
